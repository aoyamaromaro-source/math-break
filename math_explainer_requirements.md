# 数学解説アプリ「MathBreak」要件定義書

## プロジェクト概要

**アプリ名（仮）:** MathBreak  
**目的:** 高校数学が苦手な生徒（偏差値45〜50程度）が、模試・東進講座の問題や解説で「なぜそうなるのか」が理解できない箇所を、その場で深掘りできるWebアプリ  
**想定ユーザー:** 高校生（家庭内利用）  
**利用形態:** ローカルまたはVercel等へのデプロイ想定。APIキーは環境変数で管理  

---

| 項目 | 採用技術 |

## 画面構成

### 画面1: インプット画面（トップ）

**機能:**
- モード選択: 「問題のみ」 / 「問題＋解説」
- 画像アップロード（スマホ撮影 or ファイル選択）
  - 対応形式: JPEG, PNG, HEIC（iPhone対応）
  - 最大2枚（問題用・解説用）
  - ドラッグ&ドロップ対応
- 「文字起こし開始」ボタン
- ローディング表示（OCR中）

**処理:**
1. アップロードされた画像をbase64エンコード
2. Claude Vision APIへ送信
3. OCR結果（テキスト）を取得
4. 画面2へ遷移

---

### 画面2: テキスト確認・編集画面

**機能:**
- OCR結果を「問題文エリア」「解説文エリア」に分けて表示
- テキスト手動修正ができる（textarea）
- 「この内容で理解する」ボタン → 画面3へ

**目的:** OCRの誤認識をユーザーが修正できるようにする

---

### 画面3: インタラクティブ解説画面（メイン）

**レイアウト:**
- 上半分: 問題文・解説テキストの表示エリア
- 下半分: AI説明パネル（チャット形式）

**テキスト表示エリアの機能:**

#### ハイライト機能（メインインタラクション）
- ユーザーが解説テキストを選択（なぞる）すると、黄色くハイライトされる
- ハイライト部分はクリックで解除可能
- 複数箇所のハイライトが可能
- ハイライトされた状態で「この部分を説明して」ボタンが出現

#### ブロック単位のインタラクション
- 数式・計算式の行は1行単位でクリック可能
- クリックすると青色でハイライト、「なぜこうなる？」ボタンが表示

---

### AI説明パネル

**トリガー:**
1. ハイライト選択 → 「この部分を説明して」ボタン押下
2. 行クリック → 「なぜこうなる？」ボタン押下
3. 自由入力テキストボックスでの質問

**AIへのシステムプロンプト（固定）:**
```
あなたは数学が苦手な高校生（偏差値45〜50程度）に教える、やさしい数学の先生です。

以下のルールを守って説明してください：
- 中学生でもわかるような言葉を使う
- 専門用語を使う場合は必ず意味を補足する
- 「なぜそうするのか」「何のためにするのか」を必ず説明する
- 式変形は1ステップずつ、省略せず書く
- 「〜だから〜する」という因果関係を明確にする
- 難しい概念はたとえ話を使って説明する
- 答えを一方的に教えるのではなく、生徒が考えられるよう誘導する
- 励ましの言葉を適度に入れる
- 長くなりすぎず、200〜400字程度を目安にする（必要なら続きを出せる）
```

**送信内容:**
- 問題文全体（コンテキスト）
- 解説全体（コンテキスト）
- ハイライトされた部分（フォーカス）
- ユーザーの質問文（任意）

**表示:**
- ストリーミング表示（逐次表示）
- 「もっと詳しく」ボタン
- 「別の言い方で」ボタン
- 「例題を出して」ボタン（ワンクリックで関連例題を生成）
- 「わかった！」ボタン（会話をリセット）

---

## データフロー

```
[画像アップロード]
    ↓
[Claude Vision API] → OCRテキスト化
    ↓
[ユーザー確認・修正]
    ↓
[テキスト表示 + ハイライトUI]
    ↓（ハイライト＋ボタン押下）
[Claude API] → 説明文生成（ストリーミング）
    ↓
[チャットパネルに表示]
    ↓（フォローアップ質問）
[Claude API] → 会話継続（履歴付き）
```

## APIリクエスト仕様

### OCR処理

```typescript
// システムプロンプト
const OCR_SYSTEM = `
あなたは数学の問題・解説の画像をテキストに変換する専門家です。
- 数式はLaTeX形式ではなく、そのまま読める形（例: x^2, √3, 1/2）で書く
- 図・グラフは「[図: ○○を示す図]」と記載する
- 解説の行番号や記号はそのまま保持する
- 問題文と解説が混在する場合は「【問題】」「【解説】」のラベルをつける
`;
```

### 解説生成

```typescript
// 会話履歴を保持してAPIコール
const messages = [
  ...conversationHistory,
  {
    role: "user",
    content: `
【問題文】
${problemText}

【解説全体】
${solutionText}

【わからない部分】
${highlightedText}

${userQuestion ? `【質問】${userQuestion}` : "この部分を偏差値45〜50の生徒にわかるように説明してください。"}
    `
  }
]
```

## UI/UX仕様

### カラーパレット
- 背景: #FAFAF8（温かみのある白）
- テキスト: #1A1A1A
- ハイライト（選択中）: #FFF176（黄色）
- ハイライト（行クリック）: #BBDEFB（水色）
- アクセント: #1565C0（落ち着いた青 → 「先生感」）
- 説明パネル背景: #E8F5E9（薄緑 → 「わかった！」感）

### フォント
- 本文: `system-ui`（読みやすさ優先）
- 数式: `monospace`（等幅で見やすく）

### レスポンシブ
- スマホ対応必須（撮影→そのまま使う想定）
- タブレット・PC対応

### アクセシビリティ
- ハイライト操作はタッチ対応（スマホで指でなぞれる）
- フォントサイズ調整ボタン（3段階）

## Claude Codeへの補足指示

- `create-next-app` から始める（TypeScript, Tailwind, App Router を選択）
- APIキーは `.env.local` で管理。サーバーサイドAPIルートのみで使用
- ストリーミングは `ReadableStream` + `StreamingTextResponse` で実装
- ハイライト機能は `window.getSelection()` を使って実装
- 画像のbase64変換は `FileReader` API を使用
- スマホのタッチ選択でもハイライトが動作するよう `touchend` イベントにも対応
- 数式の見やすさのためにコードブロック（`<pre>`）を活用
- エラーハンドリング: API失敗時は「もう一度試してね」のメッセージを表示
